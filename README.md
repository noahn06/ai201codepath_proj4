# provenance guard

backend api that classifies whether submitted text was written by a human or generated by ai, using two independent detection signals, confidence scoring, transparency labels, and an appeals workflow.

---

## Stack

| Component | Tool |
|---|---|
| API framework | Flask 3 |
| Rate limiting | Flask-Limiter |
| Detection signal 1 | Groq (llama-3.3-70b-versatile) |
| Detection signal 2 | Stylometric heuristics (pure Python) |
| Audit log | SQLite (built-in) |
| Config | python-dotenv |

---

## Architecture

text submitted → rate limiter → llm classifier + stylometric analyzer → confidence scorer → label → audit log → response

appeal submitted → rate limiter → appeal handler → audit log → response

---

## Detection Signals

### Signal 1 — LLM Classifier (Groq)
sends the submitted text to `llama-3.3-70b-versatile` with a prompt asking for an ai probability score. returns `llm_score` ∈ [0, 1].

- **captures**: voice, word choice, weirdly smooth formatting, and general ai phrasing.
- **blind spot**: ai text prompted to use slang, typos, or messy casual structures.

### Signal 2 — Stylometric Analyzer (pure Python)
computes four structural metrics in python, each returning a score ∈ [0, 1] (1 = ai-like), averaged into `stylo_score`:

1. **sentence-length variance** — ai has lower variance because its sentence lengths are super uniform.
2. **type-token ratio** — vocab diversity; ai tends to repeat common words more at scale.
3. **punctuation density** — ai text uses punctuation way more sparingly.
4. **average sentence length** — ai text gravitates toward a ~18-word sweet spot.

- **captures**: statistical structure that the llm might not look at directly.
- **blind spot**: short texts (less than 3 sentences) because variance stats break; very polished human writing.

### Confidence Scoring

```
combined     = 0.65 * llm_score + 0.35 * stylo_score
agreement    = 1 - |llm_score - stylo_score|
polarization = 2 * |combined - 0.5|
confidence   = 0.50 * polarization + 0.50 * agreement
```

llm gets 0.65 weight because it's a stronger signal. the agreement penalty lowers confidence if signals disagree, which keeps the system from returning high confidence labels when they point in different directions.

## Transparency Labels

three label variants, chosen by confidence and combined score:

| Condition | Label variant |
|---|---|
| confidence ≥ 0.55 AND combined ≥ 0.6 | `likely_ai` |
| confidence ≥ 0.55 AND combined ≤ 0.4 | `likely_human` |
| anything else | `uncertain` |

**likely_ai:**
> "This content shows strong indicators of AI generation (high confidence). This is an automated assessment — the creator may appeal if this is incorrect."

**likely_human:**
> "This content shows strong indicators of human authorship (high confidence). This is an automated assessment."

**uncertain:**
> "The system could not confidently determine whether this content was written by a human or AI. This may happen with short texts, edited prose, or content that blends both styles. The creator may appeal if this classification is incorrect."

---

## Rate Limits

| Endpoint | Limit | Reasoning |
|---|---|---|
| POST `/submit` | 10/minute, 100/day per IP | normal creators submitting their own work shouldn't need more than 10 per minute. the 100/day cap stops scripts from scraping or spamming. |
| POST `/appeal` | 5/hour per IP | appeals require manual review so they should be rare; 5 per hour is plenty for normal creators and blocks spam. |

## Validation of Confidence Scoring

I checked if the confidence scoring actually makes sense by testing it on 4 types of text:
1. **clearly ai**: high llm score (0.80) and medium stylometric score (0.52). the high polarization and signal agreement returned confidence of `0.5581` with a `likely_ai` label.
2. **clearly human (casual)**: low llm score (0.23) and medium stylometric score (0.46). returned `likely_human` (confidence `0.5718`).
3. **formal human writing**: high llm score (0.72) and medium stylometric score (0.49). signals were moderately mismatched, so agreement score dropped, with a confidence of `0.5277` meaning it is `uncertain`
4. **lightly edited ai**: medium llm score (0.42) and medium stylometric score (0.50). lack of polarization pushed confidence down to `0.5139`, labeling as `uncertain`.

## Known Limitations

the system will probably get it wrong on **short poetry (under 30 words)**.
- **why**: 
  - *stylometric signal*: calculating sentence variance on a 1-2 sentence poem doesn't work well mathematically. also, poetry has weird punctuation rates and diverse vocab that breaks typical prose stats, so the ttr and punctuation metrics will be off.
  - *llm classifier*: with barely any text, the llm doesn't have enough semantic cues or voice to work with, so it just returns a middle score near 0.5.
  - *outcome*: these texts will almost always get pushed to `uncertain`.

---

## Spec Reflection

- **how the spec helped**: defining the uncertainty representation beforehand in `planning.md` made sure I didn't just write a simple binary classifier. it forced the planning of a wide "uncertain" middle band to prevent false positives on polished human writing.
- **where it diverged**: the spec originally said confidence had to be `0.75` for high-confidence labels. but when testing, even small signal disagreements (like llm = 0.80, stylo = 0.52) dragged agreement down, meaning the system would almost never hit the high-confidence labels. I calibrated it down to `0.55` in the code so it works on actual signal distributions.

---

## AI Usage

I used AI in two main places:
1. **groq prompt helper**: I had the AI write a json parser for the groq response. it suggested a strict json loader. I overrode this by adding an extra `try-except` block to parse raw floats if the model just returns a number instead of json, keeping the app from crashing.
2. **stylometrics logic**: the AI generated a standard type-token ratio (ttr) function. but ttr skews super high on short texts. I revised the logic to add a length-correction factor for texts over 30 words so the comparison is fair.
